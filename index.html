🛡️ 3. Drawdown & Loss 규칙

Max Loss Limit (MLL) — EOD Trailing Drawdown

쉽게 풀어서 설명: EOD Trailing Drawdown이란?

"EOD(End-of-Day) Trailing"이란, 장 마감 시점(미국 ET 오후 5시 = 한국시간 오전 6시)의 종가만 기준으로 최대 손실 한도(MLL)가 움직인다는 뜻입니다.

예를 들어, 50K 계정의 MLL이 $2,000이면 시작 시 최소 잔고는 $48,000입니다. 장중에 큰 수익이 났다가 다시 빠져도, 장이 끝날 때의 잔고만 봅니다. 장중에 잔고가 $53,000까지 올랐다가 $50,500으로 끝나면, MLL은 $50,500 - $2,000 = $48,500이 됩니다.

핵심: 장중 스윙으로는 MLL이 올라가지 않으니, 장중 고점 찍고 빠져도 바로 Breach 되지 않습니다. 다른 프랍펌의 "실시간(Intraday) Trailing"보다 훨씬 유리합니다.

  • 업데이트 시점: 매일 5:00 PM ET (= 한국시간 오전 6:00) 종가 기준
  • ITB(Initial Trail Balance) 초과 시: MLL이 영구 고정됨 — 더 이상 위로 올라가지 않음
  • MLL 브리치: 잔고가 MLL 아래로 떨어지면 → 계정 즉시 종료 (복구 불가)

Daily Loss Limit (DLL)

쉽게 풀어서 설명: DLL이란?

하루 동안 잃을 수 있는 최대 금액입니다. MLL이 "전체 계정 수명" 동안의 한도라면, DLL은 "오늘 하루"의 한도입니다.

DLL을 터치하면 당일 거래만 정지됩니다 — 계정이 날아가는 게 아니라 다음 세션부터 다시 거래할 수 있습니다. 단, DLL과 MLL을 동시에 터치하면 계정 종료입니다.

LucidScale DLL

쉽게 풀어서 설명: LucidScale DLL이란?

잔고가 Initial Trail Balance(ITB)를 넘어서면, 기존 고정 DLL 대신 최고 EOD 수익(profit) × 60%가 새로운 DLL이 됩니다.

공식: Highest end-of-day account profit × 60% = LucidScale DLL

예: 50K 계정에서 ITB($52,100)를 넘기고 최고 EOD 수익이 $4,000이라면 → $4,000 × 60% = $2,400이 새 DLL. 기존 고정 DLL $1,200보다 2배 넓어진 것입니다.

핵심: LucidScale DLL은 한번 올라가면 절대 내려가지 않습니다 — 계정이 drawdown 되더라도 최고점 기준 유지. 수익이 쌓일수록 하루 허용 손실폭이 넓어지는 트레이더 친화적 구조입니다.

📐 4. 일관성(Consistency) 규칙 상세

쉽게 풀어서 설명: Consistency Rule이란?

"하루 대박 쳐서 목표 달성"하는 것을 방지하기 위한 규칙입니다. 특정 하루의 수익이 전체 수익에서 차지하는 비중에 상한을 둡니다.

산식: 최대 단일일 수익 ÷ 전체 누적 수익 × 100 = Consistency %

예시 (40% 룰): 전체 수익이 $5,000이고, 그 중 가장 수익이 컸던 하루가 $3,000이면 → 3,000 / 5,000 = 60% → 위반! 40% 이하가 되려면 전체 수익이 최소 $7,500 이상이어야 합니다 (3,000 / 7,500 = 40%).

즉, 최고일 수익의 2.5배 이상을 전체로 벌어야 출금 가능합니다 (40% 룰 기준).

계정 유형 단계 Consistency 한도 의미
Flex Evaluation 50% 최고일 수익 ≤ 전체 수익의 절반 → 최소 2일 이상 수익일 필요
Flex Funded 없음 Funded 이후에는 Consistency 제한 없이 자유 거래

Pro Funded 40% 최고일 수익의 2.5배 이상을 전체로 벌어야 출금

Black Funded 40% Legacy, Pro와 동일
주의사항
  • 손실일은 Consistency를 악화시킵니다. 전체 수익(분모)이 줄어들면서 최대일 수익(분자)은 유지되기 때문에 비율이 올라갑니다.
  • 매 Payout 승인 후 비율이 리셋됩니다 — 새 사이클은 $0부터 다시 시작.
  • Consistency를 초과했다면 추가 거래로 전체 수익을 키워 비율을 희석해야 합니다.
실전 예시: LucidDirect 20% Consistency

월요일에 $2,000 수익 → 이것만으로 Consistency = 100% (위반).

화~금 나머지 4일 동안 합계 $8,000 이상을 더 벌어야 → 전체 $10,000 중 $2,000 = 20%로 충족.

20% 룰은 매우 빡빡합니다. 매일 비슷한 수준의 수익을 내야 한다는 의미이므로, 뉴스 트레이딩이나 큰 변동성 날에 몰아서 수익 내는 스타일과는 맞지 않습니다.

쉽게 풀어서 설명: Payout Buffer란?

출금을 요청하려면 수익이 Buffer Balance를 초과해야 합니다. Buffer = 초기 Max Loss Limit + $100입니다.

예: 50K 계정 → MLL은 $50,000 + $2,000 = $52,000에서 시작, Buffer는 $52,100. 즉, 잔고가 $52,100 이상이어야 출금 가능하고, Buffer 금액 자체에서는 출금할 수 없습니다. 출금 후에도 잔고가 Buffer 아래로 떨어지면 요청이 거부될 수 있습니다.

🔄 6. New Live Structure (2026년 업데이트)

쉽게 풀어서 설명: 왜 바뀌었나?

기존에는 Escrow Matching(회사가 수익 일부를 매칭), 60일 Vesting(수익 락업), Safety Net Account, Bonus Doubling 등 복잡한 구조가 있었습니다. 2026년 업데이트로 이 모든 것을 폐지하고, 단순한 "$0 시작 + 일회성 보너스" 구조로 바꿨습니다.

Live 전환 대상 (리뷰풀 진입 조건)

Lucid 리스크팀이 지속적으로 트레이더 성과를 모니터링합니다. 아래 조건 중 하나 이상 충족 시 Live 리뷰풀에 진입:

  • 해당 플랜의 최종 Payout(Payout 5) 수령 후
  • 누적으로 상당한 금액을 수령한 경우
  • Sim Funded에서 뛰어난 성과를 보인 경우
  • 이전에 Live 전환 경험이 있는 경우
주의
Payout 5는 최대 Payout 레벨이지, Live 전환이 보장되는 최소 조건이 아닙니다. 모든 Live 전환은 Lucid 리스크팀의 재량입니다.

Live 전환 시 구조

  • 자격 있는 Funded 계정 1개당 Live 계정 1개 발급 (각 Funded 계정에서 최소 1회 Payout 필요)
  • Live 거래 20일 후, 여러 Live 계정을 하나로 합칠 수 있는 옵션 제공 가능
  • Live 전환 시 모든 Sim 계정은 폐쇄됨 — 리저브 계정 보유 시 주의!

🚫 8. 금지 행위 (Prohibited Activities)

1. Microscalping (마이크로스캘핑)

쉽게 풀어서 설명: Microscalping이란?

극단적으로 짧은 시간(몇 초) 내에 아주 큰 사이즈로 미세한 가격 변동을 잡는 전략입니다. 이것은 실제 시장에서 작동하는 전략이 아니라, 시뮬레이션 환경에서 체결 방식의 허점을 이용하는 것입니다. 실제 시장에서는 슬리피지와 유동성 문제로 불가능한 전략이므로 금지됩니다.

  • 탐지 기준: 전체 수익의 50% 이상이 5초 이하 보유 거래에서 발생 시 플래그
  • 정상 스캘핑은 허용: 현실적 시장 활동을 반영하는 빠른 매매는 OK
  • 핵심 판단: "시뮬레이션 체결 방식을 악용하려는 의도"가 있는가

Microscalping 제재 절차 (공식)

  1. 플래그 → Lucid가 수동 리뷰로 정책 위반 여부 확인
  2. 악의적 행위 확인 시 → 서면 경고
  3. 경고 후에도 계속되면 → Microscalping으로 발생한 모든 수익 몰수 + 영구 이용 제한
  4. 오탐으로 판단 시 이의 제기(appeal) 가능

2. Hedging (헤징)

쉽게 풀어서 설명: 금지되는 Hedging이란?

같은 상품(예: NQ)에서 서로 다른 계정으로 반대 포지션을 잡는 것입니다. Account A에서 NQ Long, Account B에서 NQ Short → 한쪽은 반드시 수익. 이렇게 하면 한 계정을 희생시키고 다른 계정에서 확실한 수익을 챙기는 "보장된 수익" 구조가 됩니다.

다른 사용자와 짜고 하는 것도 금지. 다른 프랍펌과 교차해서 하는 것도 금지입니다.

허용 vs 금지 정리

상황허용?
같은 계정 내에서 같은 상품 Mini Long + Micro Short✅ 허용
같은 계정 내에서 다른 상품 Long/Short (예: ES Long + NQ Short)✅ 허용
다른 계정에서 같은 상품 Long/Short (Mini든 Micro든)🚫 금지
다른 계정에서 상관관계 있는 상품 Long/Short (예: ES Long in A + NQ Short in B)🚫 금지
다른 사용자와 짜고 반대 포지션🚫 금지
다른 프랍펌과 교차 반대 포지션🚫 금지
상관관계 상품 그룹 (Correlated Assets)
주식지수(ES, NQ, YM, RTY), 금속(GC, SI), 에너지(CL, NG) 등 같은 그룹 내 상품은 "상관관계 있음"으로 간주됩니다. 다른 계정 간에 이 그룹 내 반대 포지션을 취하면 Hedging으로 탐지됩니다.

탐지: 자동 상관관계 탐지 시스템 운영

3. High-Frequency Trading (HFT, 고빈도 매매)

쉽게 풀어서 설명: HFT란?

알고리즘이 초/밀리초 단위로 대량의 주문을 제출·실행하여 미세한 가격 비효율을 포착하는 전략입니다. 수분 내에 수백 개의 주문이 발생하여 플랫폼 인프라에 과도한 부하를 줍니다.

  • 1차 적발: 서면 경고
  • 반복: HFT로 발생한 모든 수익 제거 + 계정 폐쇄 + 영구 이용 제한
  • 오탐으로 판단되면 이의 제기(appeal) 가능

4. 시스템 악용 (Platform Exploitation)

  • 플랫폼 메커니즘이나 시뮬레이션 시장 조건을 이용한 전략 금지
  • 시스템 오류나 데이터 업데이트 지연을 이용한 행위 금지
  • "실제 시장에서 재현 불가능한 방식으로 수익을 내는 모든 행위"가 대상

위반 시 제재 (유형별 차이)

위반 유형1차반복
Hedging이메일 통보 + 전일 잔고 리셋관련 계정 전부 Breach + 영구 밴
Microscalping수동 리뷰 → 서면 경고수익 몰수 + 영구 밴
HFT서면 경고수익 제거 + 계정 폐쇄 + 영구 밴

모든 유형에서 오탐이라고 판단되면 이의 제기(appeal)가 가능합니다.
